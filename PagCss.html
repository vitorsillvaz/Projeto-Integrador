HÍSTORIA DO CSS

A história do CSS (Cascading Style Sheets) começa no início da década de 1990, quando a web ainda era uma tecnologia emergente. Naquela época, o HTML, criado por Tim Berners-Lee, tinha como principal objetivo estruturar documentos e disponibilizá-los na internet. As páginas eram simples, compostas basicamente de texto e links, com estilos limitados e feitos diretamente no HTML, utilizando tags como <font> para alterar o tamanho da fonte ou <b> para negrito. Isso resultava em uma mistura de conteúdo e formatação que dificultava a manutenção das páginas.

Em 1994, Håkon Wium Lie, trabalhando no CERN, propôs a ideia do CSS como uma forma de separar o conteúdo da apresentação. Bert Bos, que também estava desenvolvendo uma linguagem de estilo semelhante, juntou-se a Håkon e juntos trabalharam na criação de um padrão. Em dezembro de 1996, o W3C (World Wide Web Consortium) publicou a primeira recomendação oficial do CSS1. Essa versão introduziu propriedades básicas, como controle de fontes, cores, espaçamento, margens e bordas. O conceito de cascata também foi introduzido, permitindo que diferentes regras de estilo fossem aplicadas em camadas hierárquicas. Com o lançamento do CSS2 em 1998, novas funcionalidades foram adicionadas, como suporte a diferentes tipos de mídia (tela e impressão), posicionamento de elementos e pseudo-elementos como :hover. No entanto, a implementação do CSS2 pelos navegadores da época, principalmente Internet Explorer e Netscape, era inconsistente, o que gerou muitos problemas para os desenvolvedores e iniciou a chamada “Guerra dos Navegadores”.

O CSS3, lançado em 2001, trouxe uma grande evolução, principalmente com sua modularização. Cada módulo podia ser desenvolvido e adotado separadamente, facilitando a implementação pelos navegadores. Essa versão trouxe avanços significativos, como bordas arredondadas, sombras, gradientes, transições e animações. Além disso, novos seletores e a possibilidade de criar layouts responsivos com media queries ajudaram a tornar os designs mais dinâmicos e flexíveis. Navegadores mais modernos, como Google Chrome e Firefox, passaram a adotar os padrões de maneira mais consistente, melhorando a experiência de desenvolvimento. Nos últimos anos, o CSS continuou evoluindo com a introdução de novas ferramentas e funcionalidades, como Flexbox e Grid Layout, que facilitaram ainda mais a criação de layouts complexos e responsivos. Recursos como variáveis CSS e suporte a temas escuros também ajudaram a modernizar a web. O futuro do CSS promete mais avanços, com a introdução de tecnologias como o CSS Houdini, que permitirá maior controle sobre o comportamento visual das páginas. O CSS se tornou uma ferramenta essencial para o desenvolvimento web, permitindo criar experiências visuais ricas e acessíveis para todos os tipos de usuários e dispositivos.

COMO O CSS FUNCIONA

O CSS (Cascading Style Sheets) funciona definindo regras de estilo que controlam a aparência de elementos HTML em uma página web. Ele atua como uma camada separada, que especifica como o conteúdo estrutural (HTML) será exibido visualmente no navegador.

1. Seleção de Elementos

O CSS começa escolhendo os elementos HTML que devem ser estilizados, usando seletores como tags, classes e IDs. Por exemplo: 

p{
 color: red; 
}

Isso muda a cor de todo o texto em elementos <p> para vermelho.

2. Declarações de Estilo

As declarações CSS definem propriedades e valores para os elementos. Por exemplo: 

p{
 font-size: 18px;
}

Isso define o tamanho da fonte em 18 pixels para todos os parágrafos.

3. Cascata e Herança

O CSS usa a cascata para decidir quais regras aplicar quando múltiplas regras se sobrepõem. A especificidade determina qual regra tem mais prioridade. Algumas propriedades, como cores e fontes, são herdadas automaticamente pelos elementos filhos.

4. Media Queries

Media queries permitem criar estilos responsivos que mudam com base no tamanho da tela: 


@media (max-width: 768px){
   body { 
   font-size: 14px;
   } 
}

5. Box Model

O CSS usa o modelo de caixa para calcular o espaço ocupado pelos elementos, incluindo conteúdo, padding, bordas e margens.

6. Pseudo-classes e Pseudo-elementos

Pseudo-classes aplicam estilos com base em estados, como hover: 

 a:hover {
  color: blue;
 }

Pseudo-elementos, como ::before e ::after, permitem adicionar conteúdo extra antes ou depois de um elemento.
